# Agent Rules 한국어 참조

> 사람 전용 한국어 참조 문서입니다. 기준 문서는 `AGENTS.md`입니다.

## 핵심 목적

이 저장소는 Home Search V1 마이그레이션 대상이다. 목표는 부동산 아파트 실거래 데이터를 수집하고, 안전하게 저장하고, 기존 주요 API URL을 유지하면서 지도에 표시하는 것이다.

빠른 기능 추가보다 데이터 정확성, 추적 가능성, API 호환성이 우선이다.

## 기준 문서

AI 에이전트는 `docs/README.md`부터 시작해 마이그레이션 계획, 아키텍처, 데이터 저장, API 계약, 지도 흐름, UI/UX, 인프라 문서를 순서대로 기준으로 삼아야 한다.

소스 백엔드와 프론트엔드는 읽기 전용 참조로만 사용하고, 실제 변경은 `/Users/gwongwangjae/home-search` 안에서만 한다.

## 작업 원칙

- 변경 전 `git status --short`를 확인한다.
- 문서와 관련 코드를 먼저 탐색한다.
- 모호하면 짧은 계획을 먼저 세운다.
- 변경은 작게 유지하고, 불필요한 리팩터를 피한다.
- 변경 후 검증한다.

## V1 보호 규칙

- V1 API URL을 유지한다.
- 랭킹, 즐겨찾기, 알림, 메일 배치, 추천, 인사이트, 무거운 분석 기능은 V2 범위다.
- UI는 바꿀 수 있지만 API 계약은 유지해야 한다.

## 데이터/API 불변 조건

- raw ingest 기록이 normalized trade보다 먼저 남아야 한다.
- 중복 수집이 normalized trade 중복을 만들면 안 된다.
- 매칭 실패는 추적 가능해야 한다.
- 운영 관계는 `complex_id` 기준이다.
- `complex_pk`, `apt_seq`, `source`, `source_key`는 감사와 매칭 보조 정보로 보존한다.

## KO 문서 규칙

`*_KO.md`는 사람 전용 한국어 참조 파일이다. 동기화 상태는 `.ko-docs.toml`에서 관리한다.

공용 `.md`를 만들거나 의미 있게 수정하면 대응하는 `*_KO.md`도 함께 갱신한다. 기존 KO 파일을 읽어 수정하지 않고, 원문 `.md` 기준으로 새로 작성한다.
